# LINE公式アカウント連携マニュアル

petit event maker では、LINE公式アカウントを連携することで以下の機能が使えるようになります。

- イベント公開時にフォロワーへ**リッチカード（画像・日時・価格付き）**で自動通知
- 新しい予約が入ったとき**LINEのDMで即時通知**
- キャンセル・キャンセル待ち繰り上げの自動通知

## 1. 事前準備

以下のものを用意してください。

| 必要なもの | 説明 |
|-----------|------|
| LINE公式アカウント | まだ持っていない場合は [LINE Official Account Manager](https://manager.line.biz/) で無料作成できます |
| LINE Developersアカウント | [LINE Developers](https://developers.line.biz/) にLINEアカウントでログインすれば自動作成されます |
| petit event makerのアカウント | ログイン済みであること |

> **ポイント**: LINE公式アカウントは無料プラン（コミュニケーションプラン）で十分使えます。月200通までのメッセージが無料です。

## 2. LINE Developersでチャネルを作成する

ここが一番つまづきやすいステップです。ゆっくり進めましょう。

### 2-1. LINE Developersコンソールにログイン

1. [LINE Developers](https://developers.line.biz/console/) にアクセス
2. 「LINEアカウントでログイン」をクリック
3. 普段使っているLINEアカウントでログイン

### 2-2. プロバイダーを作成（初回のみ）

1. 左サイドバーの「プロバイダー」をクリック
2. 「作成」ボタンをクリック
3. プロバイダー名を入力（例：「自分の名前」や「お店の名前」）
4. 「作成」をクリック

> **注意**: プロバイダーは「アプリの開発者」を表すものです。ユーザーに公開されます。

### 2-3. Messaging APIチャネルを作成

1. 作成したプロバイダーの画面で「Messaging API」をクリック
2. 以下の情報を入力:

| 項目 | 入力内容 |
|------|---------|
| チャネル名 | LINE公式アカウントの名前（例：「○○イベント」） |
| チャネル説明 | 簡単な説明文 |
| 大業種 | 該当するものを選択 |
| 小業種 | 該当するものを選択 |
| メールアドレス | 連絡先メールアドレス |

3. 利用規約に同意して「作成」をクリック

> **既にLINE公式アカウントをお持ちの場合**: LINE Official Account ManagerからMessaging APIを有効にする方法もあります。その場合は [LINE Official Account Manager](https://manager.line.biz/) → 設定 → Messaging API → 「Messaging APIを利用する」から設定できます。

## 3. チャネルアクセストークンとシークレットを取得する

### 3-1. チャネルシークレットを取得

1. LINE Developersコンソールで、作成したチャネルを開く
2. **「チャネル基本設定」**タブをクリック
3. ページ下部の「チャネルシークレット」を確認
4. **コピーボタン**でコピー（メモ帳などに一時保存）

### 3-2. チャネルアクセストークンを取得

1. **「Messaging API設定」**タブをクリック
2. ページ最下部にある「チャネルアクセストークン（長期）」のセクションを見つける
3. **「発行」ボタン**をクリック
4. 表示されたトークンを**コピーボタン**でコピー（メモ帳などに一時保存）

> **よくある間違い**:
> - 「チャネル基本設定」タブにあるのが**シークレット**
> - 「Messaging API設定」タブにあるのが**アクセストークン**
> - この2つを取り違えないように注意してください

> **重要**: チャネルアクセストークンは非常に長い文字列です。途中で切れないよう、必ずコピーボタンを使ってください。

## 4. petit event makerで接続テストする

1. petit event makerにログイン
2. 右上のメニュー → **「設定」** → **「LINE連携設定」** を開く（またはURL: `/settings/line`）
3. 「チャネルアクセストークン」の欄にコピーしたトークンを貼り付け
4. 「チャネルシークレット」の欄にコピーしたシークレットを貼り付け
5. **「接続テスト＆保存」**ボタンをクリック

### 成功した場合

- 画面が切り替わり、LINE公式アカウント名が表示されます
- 「接続済み」のバッジが表示されます
- → 次のステップ（Webhook URL設定）に進んでください

### 失敗した場合

- エラーメッセージを確認してください
- よくある原因は [トラブルシューティング](#8-よくあるトラブルと対処法) を参照

## 5. Webhook URLを設定する

**このステップを忘れると、友だち追加の自動検知やフォロワー管理ができません。**

### 5-1. Webhook URLをコピー

1. 接続成功後の画面に「Webhook URL」が表示されます
2. URLの右にある**コピーボタン**をクリック

### 5-2. LINE DevelopersでWebhook URLを設定

1. [LINE Developersコンソール](https://developers.line.biz/console/) に戻る
2. チャネルを選択 → **「Messaging API設定」**タブを開く
3. 「Webhook設定」セクションを見つける
4. **「Webhook URL」**の「編集」をクリック
5. コピーしたURLを貼り付け
6. **「更新」**をクリック
7. **「Webhookの利用」をON**にする（トグルスイッチを有効にする）

### 5-3. 検証する

- 「検証」ボタンをクリックして「成功」と表示されればOKです

> **よくある間違い**:
> - URLを貼り付けたあと「更新」を押し忘れる
> - **「Webhookの利用」をONにし忘れる** ← これが一番多いミスです
> - URLの前後に余計なスペースが入っている

## 6. LINE公式アカウントを友だち追加する

Webhook設定が完了したら、**あなた自身が**LINE公式アカウントを友だちに追加します。

### 6-1. LINE公式アカウントのQRコードを表示

1. LINE Developersコンソール → **「Messaging API設定」**タブ
2. 「ボット情報」セクションにQRコードが表示されています
3. スマートフォンのLINEアプリでQRコードを読み取る

### 6-2. 友だち追加する

1. LINEアプリでQRコードを読み取る
2. 「追加」をタップして友だちに追加

### 6-3. petit event makerで確認

1. petit event makerの「LINE連携設定」ページに戻る
2. ページをリロード（更新）する
3. **「フォロワー一覧」にあなたの名前が表示されていれば成功**です

> **フォロワーが表示されない場合**:
> - Webhook URLが正しく設定されているか確認
> - 「Webhookの利用」がONになっているか確認
> - 一度ブロックしてブロック解除すると再検知されます

---

## 7. 通知先を設定する

フォロワー一覧に自分が表示されたら、**通知先に設定**します。

1. フォロワー一覧で自分の名前の横にある **「通知先に設定」** ボタンをクリック
2. ボタンが **「通知先」** バッジに変わったら設定完了

### 通知先を設定すると

| 設定前 | 設定後 |
|-------|-------|
| 予約通知がブロードキャスト（全員に配信） | 予約通知が**1:1のDM**であなたに届く |

> **ポイント**: 通知先を設定することで、予約通知がフォロワー全員に送信されるのを防ぎ、あなた（イベント主催者）だけに届くようになります。

---

## これで設定完了です

以下の機能が使えるようになりました：

| 機能 | 動作 |
|------|------|
| イベント公開通知 | イベントを公開するとき、フォロワーにリッチカードで通知 |
| 予約通知（主催者） | 新しい予約が入ると、あなたにDMで通知 |
| 予約確認（参加者） | 予約した人にリッチカードで確認通知 |
| キャンセル通知 | キャンセル時に主催者と参加者に通知 |
| 繰り上げ通知 | キャンセル待ちの人が繰り上がったとき自動通知 |

## 8. よくあるトラブルと対処法

### Q1. 「接続テスト＆保存」でエラーになる

| 考えられる原因 | 対処法 |
|-------------|--------|
| トークンが間違っている | 「Messaging API設定」タブからコピーし直してください。**チャネル基本設定のシークレットと間違えていないか**確認 |
| トークンが途中で切れている | コピーボタンを使って確実にコピーしてください。手動選択だと途中で切れることがあります |
| シークレットが間違っている | 「チャネル基本設定」タブからコピーし直してください |
| チャネルが無効になっている | LINE Developersコンソールでチャネルのステータスが「公開」になっているか確認 |

### Q2. Webhook URLの「検証」が失敗する

| 考えられる原因 | 対処法 |
|-------------|--------|
| URLが正しくコピーされていない | petit event makerの画面からもう一度コピーしてください |
| URLの前後にスペースが入っている | 余計なスペースを削除してください |
| httpsでない | URLは `https://` で始まっている必要があります（petit event makerでは自動的にhttpsです） |

### Q3. 友だち追加したのにフォロワー一覧に表示されない

| 考えられる原因 | 対処法 |
|-------------|--------|
| Webhook URLが未設定 | [ステップ5](#5-webhook-urlを設定する) を再確認 |
| 「Webhookの利用」がOFF | LINE Developersコンソール → Messaging API設定 → 「Webhookの利用」をONに |
| 応答設定の問題 | LINE Official Account Manager → 設定 → 応答設定 → 「Webhook」をONに |
| ページを更新していない | ブラウザをリロードしてください |

> **裏技**: フォロワーが検知されない場合、一度LINEアプリでその公式アカウントを**ブロック→ブロック解除**してみてください。ブロック解除時にWebhookが再送信されます。

### Q4. 予約通知が届かない

| 考えられる原因 | 対処法 |
|-------------|--------|
| 通知先が未設定 | [ステップ7](#7-通知先を設定する) でフォロワー一覧から「通知先に設定」してください |
| 通知がOFF | LINE連携設定ページで「新規予約のLINE通知」がONになっているか確認 |
| LINE公式アカウントをブロックしている | LINEアプリで友だちリストを確認し、ブロックしていないか確認 |

### Q5. LINE Developersのどこにアクセストークンがあるか分からない

以下の順番でたどってください：

```
LINE Developersコンソール
  → プロバイダーを選択
    → チャネルを選択
      → 「Messaging API設定」タブ（※「チャネル基本設定」ではない）
        → ページの一番下までスクロール
          → 「チャネルアクセストークン（長期）」
            → 「発行」ボタン
```

### Q6. LINE Official Account Managerとの違いが分からない

| サービス | URL | 用途 |
|---------|-----|------|
| LINE Official Account Manager | manager.line.biz | LINE公式アカウントの管理（メッセージ配信、友だち管理など） |
| LINE Developers | developers.line.biz | 開発者向け設定（API設定、Webhook、トークン発行） |

petit event makerの連携に必要な操作は**LINE Developers**で行います。

### Q7. 無料プランでも使えますか？

はい、使えます。LINEの無料プラン（コミュニケーションプラン）では月200通のメッセージが送信できます。1通あたり1つの吹き出し（Flex Message含む）としてカウントされます。

```
① LINE Developersでチャネル作成
        ↓
② アクセストークンとシークレットを取得
        ↓
③ petit event makerで接続テスト
        ↓
④ Webhook URLをLINE Developersに設定 ← 忘れがち！
        ↓
⑤ LINE公式アカウントを友だち追加
        ↓
⑥ フォロワー一覧から「通知先に設定」
        ↓
    🎉 設定完了！
```
