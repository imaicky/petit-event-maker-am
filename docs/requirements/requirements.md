# petit-event-maker 要件定義書

**バージョン**: 1.0.0
**作成日**: 2026-05-08
**対象**: petit-event-maker
**位置づけ**: Peatix上位互換 × AIコンテンツホルダー特化プラットフォーム

---

## 1. プロダクトビジョン

### 1.1 一文ビジョン
> 個人のAI教育者が **集客・運営・ファン化** を一気通貫で行え、参加者の興味データが運営者に還元され続ける、リアル/オンライン/ハイブリッドを問わないイベント基盤。

### 1.2 3本柱

| 柱 | 中身 | 判断軸 |
|---|---|---|
| 1. 運営負荷削減 | イベント運営のあらゆる工程をAI/自動化で軽減 | 「主催者が楽になるか」 |
| 2. 資産化 | 主催者・参加者の活動が継続的に積み上がる | 「データ・関係が次回に活きるか」 |
| 3. レコメンド | 参加者の傾向解析→興味に刺さるイベント自動配信 | 「集客力を増幅するか」 |

### 1.3 差別化ポジション

| 軸 | Peatix | petit-event-maker |
|---|---|---|
| 対象 | 汎用イベント | **AIコンテンツホルダー特化** |
| 開催形式 | 主にリアル | **リアル/オンライン/ハイブリッド均等** |
| LINE連携 | 弱い | **深い統合（公式アカウント・配信・タグ）** |
| レコメンド | 標準 | **興味プロファイルベースのパーソナライズ** |
| ファン化 | フォロー機能あり | **フォロー＋継続収益基盤** |
| 収益モデル | 手数料 | **手数料 + 月額 + フリーミアム（全方位）** |

## 2. ターゲットペルソナ

### 2.1 メインペルソナ：個人のAI教育者

| 項目 | 内容 |
|---|---|
| 職業 | 個人で活動するAI教育者（フリーランス・副業含む） |
| 例示 | プロンプトエンジニアリング講師 / 生成AI入門講座主催者 / AIツール活用Tipsクリエイター |
| 規模 | 1〜数名運営、参加者規模 5〜50名/回 |
| 主な活動 | ワークショップ、勉強会、ハンズオン、座談会、コミュニティ運営、コンテンツ販売 |
| 課題 | 集客チャネルが分散（X/LINE/Peatix/Connpass） / 参加者の興味が次回に活かせない / リピーター化が手作業 / 告知文/LP作成が重い / 当日運営の手間 |
| ニーズ | 一元的な集客・運営・ファン化基盤、参加者データの可視化、リピート誘導の自動化 |

### 2.2 サブペルソナ

| ペルソナ | 概要 |
|---|---|
| 参加者（学習者） | 生成AI/プロンプト/AIツールに興味のある社会人・学生。学びを継続したい |
| 共同主催者 | 主催者をサポートするコラボレーター（既存`event_admins`の延長） |

## 3. 想定イベント形式

| 形式 | 比率 | 例 |
|---|---|---|
| リアル | ~33% | 対面ワークショップ、勉強会、もくもく会 |
| オンライン | ~33% | Zoom/YouTube Live配信講座、ウェビナー |
| ハイブリッド | ~33% | 会場+配信、アーカイブ販売付き |

**設計方針**: いずれの形式も **一級市民** として扱う。形式に応じて必要な機能（会場情報 / 配信URL / アーカイブ / QR受付 / オンラインチャットなど）を切替可能にする。

## 4. 収益モデル（全方位）

### 4.1 3層構造

| レイヤー | 主催者向けプラン | 参加者向け |
|---|---|---|
| **Free** | 月3イベントまで・基本機能 | 検索・予約・通知 |
| **Pro（月額）** | 無制限イベント・AI生成・レコメンド・ポートフォリオ・分析ダッシュボード | プレミアム検索・お気に入り無制限 |
| **手数料** | 有料イベントの売上から % 課金（無料イベントは無料） | — |

### 4.2 設計原則
- **フリーミアム成立**: 無料層でも「主催・参加・基本機能」は完結する
- **Proで差をつける**: AI支援・レコメンド・分析は有料層の主役
- **手数料は重複しない**: 月額Pro加入者は手数料率を優遇/または無料化（要検証）

> ⚠️ 具体的なプラン金額・手数料率は別途ビジネス検討（要件定義スコープ外）

## 5. 機能要件

### 5.1 既存機能（継続/強化）

| 領域 | 機能 | 状態 |
|---|---|---|
| イベント | 単発イベント（events） | ✅ 継続 |
| イベント | 定期メニュー（menus） | ✅ 継続 |
| 予約 | カスタムフィールド | ✅ 強化（テンプレ拡充） |
| 予約 | キャンセル待ち自動繰上げ | ✅ 継続 |
| 決済 | Stripe + マルチ決済 | ✅ 継続 |
| 通知 | LINE公式アカウント連携 | ✅ 強化（タグ拡充） |
| 通知 | メール（リマインダー/通知） | ✅ 継続 |
| 限定公開 | invite token / passcode | ✅ 継続 |
| 評価 | reviews / ratings | ✅ 強化（テキスト分析） |
| プロフィール | カスタムユーザー名 / SNS links | ✅ 継続 |
| 解析 | PostHog | ✅ 強化（特徴量化） |

### 5.2 新規機能（3本柱別）

#### 柱1: 運営負荷削減

| ID | 機能 | 優先度 | 概要 |
|---|---|---|---|
| F1-01 | AI生成（タイトル・説明・告知文） | P1 | Claude API利用、テンプレベース＋AI補完 |
| F1-02 | AI生成サムネイル | P1 | 画像生成API連携（候補：Imagen/SDXL/Anthropic画像出力） |
| F1-03 | 当日QR受付フロー | P1 | モバイル受付UI、参加者QRチェックイン |
| F1-04 | 領収書/請求書自動発行 | P2 | PDF生成、Stripe Receipts連携 |
| F1-05 | アンケート自動収集 + AI要約 | P2 | イベント後フォーム→Claude要約→主催者ダッシュボード |
| F1-06 | イベント運営テンプレ拡充 | P2 | AI教育者向けテンプレ（プロンプト講座/生成AI入門等） |
| F1-07 | 参加者横断管理 | P2 | 主催者の全イベントを跨いだ参加者管理ビュー |

#### 柱2: 資産化

| ID | 機能 | 優先度 | 概要 |
|---|---|---|---|
| F2-01 | 主催者フォロー機能 | **P0** | プラットフォーム内フォロー、新規イベント自動通知 |
| F2-02 | 参加者ダッシュボード | P1 | 参加履歴・参加証明・タイムライン、`/my` 拡張 |
| F2-03 | 主催者ポートフォリオページ | P1 | 過去開催実績・参加者の声集約、`/[username]` 拡張 |
| F2-04 | 学び資料DLボックス | P2 | イベント毎に資料添付・参加者限定DL |
| F2-05 | 参加者同士の繋がり | P3 | 同意ベースの名刺交換（プロフィール公開許可制） |
| F2-06 | プラットフォーム内DM | P3 | 主催者⇔参加者の個別メッセージ |
| F2-07 | アフターコミュニティ動線 | P3 | 次回告知・継続講座への接続UI |

#### 柱3: レコメンド

| ID | 機能 | 優先度 | 概要 |
|---|---|---|---|
| F3-01 | AI領域カテゴリ/タグ再設計 | **P0** | LLM活用 / 画像生成 / 自動化 / プロンプト等のタクソノミー |

## 6. 非機能要件（概要）

> 詳細は `non-functional.md` 参照

| 項目 | 基準 |
|---|---|
| パフォーマンス | LCP < 2.5s, FCP < 1.8s |
| アクセシビリティ | WCAG 2.1 AA |
| 可用性 | 99.5%以上（Vercel SLA準拠） |
| セキュリティ | Supabase RLS全テーブル / OWASP Top 10対応 |
| プライバシー | 参加者データの扱いは同意ベース、GDPR/個人情報保護法準拠 |
| スケール | 同時1,000ユーザー、月間100万PV想定 |

## 7. 制約条件

| 項目 | 内容 |
|---|---|

## 8. スコープ外（現時点）

- 主催者の組織化（チーム機能）
- B2B向け企業研修プラットフォーム
- 動画配信機能の自前実装（Zoom/YouTube Liveに依存）
- ネイティブモバイルアプリ（PWA対応で代替）
