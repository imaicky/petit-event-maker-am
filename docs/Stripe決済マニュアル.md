# Stripe決済 セットアップマニュアル

petit event maker では、Stripeを連携することで以下の機能が使えるようになります。

- 有料イベントの参加費を**クレジットカードで安全に受付**
- 決済完了後に**自動で「支払済」ステータスに更新**
- キャンセル時の**自動返金処理**
- 参加者一覧で**支払いステータスを一覧管理**

## 1. 事前準備

以下のものを用意してください。

| 必要なもの | 説明 |
|-----------|------|
| メールアドレス | Stripeアカウント作成用 |
| petit event makerのアカウント | ログイン済みであること |

> **ポイント**: Stripeは初期費用・月額費用なし。決済時のみ手数料（3.6%）がかかります。テストモードでは実際のお金は動きません。

---

## 2. Stripeアカウントを作成する

1. [Stripe登録ページ](https://dashboard.stripe.com/register) にアクセス
2. メールアドレス、名前、パスワードを入力
3. 「アカウントを作成」をクリック
4. メールアドレスの確認メールが届くので、リンクをクリックして確認

> **既にStripeアカウントをお持ちの場合**: [ダッシュボードにログイン](https://dashboard.stripe.com/login) してステップ3に進んでください。

> **ビジネス情報の入力**: テストモードで使う場合は後回しにできます。本番決済を開始する前には完了が必要です。

---

## 3. シークレットキーを取得する

### 3-1. テストモードに切り替え

1. Stripeダッシュボードにログイン
2. 左サイドバーの一番下にある「**開発者**」をクリック
3. 「開発者」の横にあるトグルスイッチを**ONに**すると、テストモードに切り替わります
4. ヘッダーにオレンジ色の「**テストモード**」バッジが表示されれば成功です

### 3-2. シークレットキーを取得

1. 「開発者」→「**APIキー**」タブを開く
2. 「**シークレットキー**」セクションを確認
3. キーが表示されていない場合は「**＋ シークレットキーを作成**」をクリック
4. キーの使用方法を選ぶ画面が表示されたら「**構築した連携を強化**」を選択
5. 表示されたキー（`sk_test_...`で始まる文字列）をコピー

> **重要**: シークレットキーは**絶対に公開しないでください**。GitHubにコミットしたり、フロントエンドのコードに含めてはいけません。

## 4. petit event makerで接続する

ここが従来との大きな違いです。**シークレットキーを1つ貼り付けるだけ**で、Webhook設定を含めすべて自動で連携されます。

### 手順

1. petit event makerにログイン
2. [Stripe決済設定ページ](/settings/stripe) を開く
3. 「**シークレットキー**」フィールドに、ステップ3でコピーしたキーを貼り付け
4. 「**接続テスト＆保存**」ボタンをクリック

### 自動で行われること

接続テスト＆保存をクリックすると、以下がすべて自動で実行されます:

| 処理 | 説明 |
|------|------|
| キーの検証 | Stripe APIにアクセスしてキーが正しいか確認 |
| Webhook自動作成 | 決済完了・返金など5種類のイベント通知を自動登録 |
| 署名シークレット取得 | Webhookの署名検証用キーを自動保存 |
| アカウント情報取得 | 表示名・テスト/本番モードの自動判定 |

> **Webhook手動設定やVercel環境変数の設定は一切不要です。**

### 接続成功の確認

接続が成功すると:
- 「接続済み」バッジが表示される
- アカウント名が表示される
- テストモード/本番モードのバッジが表示される
- 「Webhookエンドポイントは自動設定済みです」と表示される

## 5. テスト決済で動作確認する

### テスト手順

1. 有料イベント（参加費 > 0円）を作成する
2. イベントページから申し込む → 「決済に進む」ボタンをクリック
3. Stripe Checkoutの画面が表示される
4. 以下のテストカード番号を入力して決済する

### テスト用カード番号

| カード番号 | 結果 |
|-----------|------|
| `4242 4242 4242 4242` | 成功する決済 |

- **有効期限**: 未来の日付なら何でもOK（例: 12/34）
- **CVC**: 3桁の数字なら何でもOK（例: 123）
- **名前・郵便番号**: 何でもOK

### 確認ポイント

- [ ] 決済完了後、サンクスページに「決済＆お申し込み完了！」と表示される
- [ ] 参加者一覧に「支払済」バッジが表示される
- [ ] Stripeダッシュボードの「支払い」に決済が記録されている

> **ポイント**: テストモードの決済は何度でも無料で試せます。実際のカードは課金されないので安心してテストしてください。

## 6. 本番モードに切り替える

テストが成功したら、本番モードに切り替えて実際の決済を開始できます。

### 手順

1. Stripeダッシュボードでビジネス情報の入力を完了する（テストモードでスキップした場合）
2. 左サイドバー下部の「開発者」横のテストモードトグルを**OFF**（本番モード）に切り替える
3. 「開発者」→「APIキー」で**本番用のシークレットキー**を取得（`sk_live_...`）
4. [Stripe決済設定ページ](/settings/stripe)で一度連携を**解除**し、本番キーで**再連携**する

> **ポイント**: 本番キー（`sk_live_...`）で再連携すると、本番用のWebhookも自動作成されます。環境変数の手動変更や再デプロイは不要です。

## 7. 返金について

イベントキャンセルや参加者都合での返金は、以下の2つの方法で行えます。

### 方法1: アプリ内でキャンセル（自動返金）

参加者一覧からキャンセルすると、支払済の予約は**自動で返金処理**が行われます。

### 方法2: Stripeダッシュボードから手動返金

1. [Stripeダッシュボード → 支払い](https://dashboard.stripe.com/payments) を開く
2. 返金したい支払いをクリック
3. 「返金」ボタンをクリック → 全額または一部返金を選択
4. 返金理由を選択して確認 → 完了

> **注意**: 返金はStripeの手数料分を含めて全額返金されます（手数料は主催者負担）。返金後、参加者のカードに5〜10営業日で返金額が戻ります。

## 8. よくあるトラブルと対処法

### Q: 「シークレットキーが無効です」と表示される

- テストモードのキー（`sk_test_...`）を使用しているか確認してください
- キーの前後に余計なスペースや改行が含まれていないか確認してください
- Stripeダッシュボードで新しいキーを作成して再試行してください

### Q: 決済ページが表示されない

- [Stripe決済設定ページ](/settings/stripe)で「接続済み」になっているか確認してください
- イベントの参加費が0円になっていないか確認してください（0円のイベントでは決済は無効）

### Q: 支払済バッジが表示されない（Webhookの問題）

Webhookは連携時に自動設定されますが、問題がある場合:
- [設定ページ](/settings/stripe)で一度連携を解除し、再連携してください（Webhookが再作成されます）
- Stripeダッシュボードの「ワークベンチ」→「Webhook」で送信状態を確認してください

### Q: テストモードと本番モードの違いは？

| 項目 | テストモード | 本番モード |
|------|------------|-----------|
| キー | `sk_test_...` | `sk_live_...` |
| 実際の課金 | なし | あり |
| テストカード | 使用可能 | 使用不可 |
| データ | テスト用データのみ | 本番データのみ |

テストモードと本番モードのデータは完全に分離されています。

### Q: 無料イベントへの影響は？

**一切ありません。** Stripeが連携されていない場合、またはイベントの参加費が0円の場合は、これまでと同じ無料の申込フローが使われます。

### Q: Stripeの手数料はいくらですか？

日本のカード決済の場合、**3.6%**の手数料がかかります。例えば1,000円のイベントなら36円が手数料として差し引かれ、964円が入金されます。初期費用・月額費用は一切ありません。

### Q: 入金サイクルは？

日本では**週1回**、登録した銀行口座に自動で振り込まれます。初回の入金にはアカウント作成から7〜14日ほどかかる場合があります。

### Q: 連携を解除するとどうなる？

- 有料イベントの新規決済が停止されます
- StripeのWebhookエンドポイントも自動削除されます
- 既に完了した決済や支払いデータには影響しません
- Stripeダッシュボードからの返金は引き続き可能です

| 項目 | 詳細 |
|------|------|
